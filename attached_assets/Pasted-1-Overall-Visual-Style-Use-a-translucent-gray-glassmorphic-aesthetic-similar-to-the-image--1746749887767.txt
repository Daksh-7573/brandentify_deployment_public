1. Overall Visual Style
Use a translucent/gray "glassmorphic" aesthetic, similar to the image.


Panels should have semi-transparent backgrounds, soft shadows, and subtle borders.


Keep everything flat and fixed to the screen (no 3D or parallax/floating behavior).


✅ Best Practices:
Use neutral grayscale backgrounds (#f3f4f6 to #1f1f1f) with subtle transparency (rgba(255,255,255,0.1–0.3)).


Apply blur effects (backdrop-filter: blur(12px)) on main containers.


Avoid harsh drop shadows; prefer soft depth via light overlays.



2. Layout Structure
Use a grid-based layout: hero section, navigation bar (vertical or top), media carousel/grid, and sidebar.


Maintain consistent spacing, use padding (24–32px) for clarity.


✅ Best Practices:
Use 12-column grid for flexibility across breakpoints.


Maintain minimum spacing of 16px between components.


Side panels or cards should align in modular fashion, not float.



3. Component Styling
🔘 Buttons:
Rounded corners (12–16px), subtle shadows.


Minimalist icons or labels.


Primary CTA should use bold, saturated colors (e.g., blue or teal) for standout.


🔲 Cards:
Glass-style cards with:


Translucent gray background


Soft inner shadows


Hover effect with a slight scale or glow


🔍 Search Bar:
Center-aligned or top-fixed.


Use placeholder animation or text suggestions for engagement.


🎚 Media Controls:
Simple, flat-style player at bottom or embedded in content.


Seek bar, play/pause, volume — all accessible and touch-friendly.



4. Typography
Use modern, clean sans-serif fonts (e.g., Inter, SF Pro, or Poppins).


Emphasize clear hierarchy:


Headings: font-weight: 600–700


Body: 400–500


Use subtle contrast (not pure black)

5. Responsiveness
Use flex or grid layouts with media queries to reflow:


Cards become stacked on mobile


Side panels collapse into overlays or drawers


Controls (like the mini player) adapt to bottom-fixed bar on mobile



6. Interactions (Non-Movable)
Since UI elements are fixed:


Use hover/focus states for visual feedback


Animate modals/panels in/out with fade + slide transitions


Make collapsible/expandable sections instead of draggable ones




8. Accessibility
Use proper color contrast (WCAG AA)


Focus indicators for keyboard navigation


Avoid hidden elements on smaller viewports (collapsible instead)
