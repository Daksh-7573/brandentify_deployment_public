PART 1 — THE 3D PORTFOLIO (Design & Styling Brief)
Concept

A premium 3D-enabled portfolio where the profile IS the product: immersive, tactile, and interactive. The page centers on a persistent 3D profile card (“3D Card”) that anchors the page while the rest of the site sections sit around it. The card invites micro-interactions (tilt, parallax, hover rings) while the site remains readable and accessible.

Visual Tone

Tech-luxury, tactile, modern.

Night-mode primary (charcoal background) with neon accents.

Focus on depth, soft glows, and elegant motion.

Color System (final)

Use the quantum palette you supplied — consistent across components.

Role	Color	HEX / rgba
Primary Accent (electric)	Electric Blue	#38bdf8
Secondary Accent	Neon Purple	#c084fc
Tertiary Accent	Mint Green	#10b981

Usage

Electric Blue = primary interactive color (CTAs, copy success, glow).

Neon Purple = accent borders, secondary badges.

Mint = success states and small badges.

Charcoal Black = full-screen background, makes neon pop.

Typography

Headings: Sora (or Inter Display heavier weight) — bold, slightly rounded.

Body: Inter — readable UI copy.

Mono accents: JetBrains Mono for small code-like chips.

Sizes:

Hero Name on card: 28px (2xl)

Section headings: 22–28px

Body: 14–16px

Micro: 12px

Motion System & Performance Rules

Respect prefers-reduced-motion: reduce tilt amplitude and remove non-essential animations.

Limit long-running GPU-heavy effects. Use will-change: transform sparingly and avoid animating box-shadow too many layers.

Use CSS transforms and opacity for smooth GPU-accelerated animations.

2) Hero + 3D Card (centerpiece)

3D Card — UX summary (core interactions)

MouseMove driven tilt: calculate rotX & rotY relative to card center (formulas below).

Parallax layers (data-layer depth 1–5) move independently with translate3d.

Hover: intensify glow & box-shadow; show profile rings and particle opacity increase.

Click on card CTA buttons: expand mini-actions (copy link, download resume, contact).

Tilt math (implement in JS):

const maxRot = 10;
const cardCenter = {x: rect.left + rect.width/2, y: rect.top + rect.height/2};
const relX = (mouseX - cardCenter.x) / (rect.width / 2); // -1 to 1
const relY = (mouseY - cardCenter.y) / (rect.height / 2); // -1 to 1
rotY = relX * maxRot; // horizontal movement rotates Y
rotX = relY * -maxRot; // vertical movement inverted for natural perspective


Apply CSS:
transform: rotateX(rotXdeg) rotateY(rotYdeg) translateZ(0);

Parallax formula per layer:
translate3d(moveX = relX * depth * 8, moveY = relY * depth * 8, 0)

Depth mapping:

Layer 5 (furthest) depth 0.25

Layer 4 depth 0.5

Layer 3 depth 0.9

Layer 2 depth 1.4

Layer 1 (closest) depth 2.2

Profile picture

Dual rings: outer spin (8s), inner pulse (3s), float (5s).

Reflection sweep: CSS gradient overlay sliding diagonally.

Implementation Hints & Developer Notes

3D interactions: implement tilt/transform with a requestAnimationFrame loop and damping for smooth return-to-zero behavior. Debounce mousemove to ~60fps. Use CSS variables to feed transforms into styles for smoother reflow.

Parallax layers: render as absolutely positioned children inside the card container; move them with transform: translate3d(...).

Particles: use CSS for small count; move to Canvas/WebGL for high counts.

Accessibility: provide non-3D fallback: when prefers-reduced-motion or keyboard-only, show static card with subtle hover states (no tilt).

Performance: use will-change: transform, opacity for animated elements; remove box-shadow animations on mobile if perf suffers.

Testing: verify tilt math on touch devices (map touchmove to lower amplitude) and reduce parallax depth on smaller screens.

Analytics: instrument CTA clicks and onAction events, copy-to-clipboard counts, project modal opens.

Quick Code Snippet — card transform (implementation pattern)
// pseudo-code
let rafId;
const card = document.querySelector('.three-d-card');
card.addEventListener('mousemove', onMove);
card.addEventListener('mouseleave', onLeave);

function onMove(e) {
  const rect = card.getBoundingClientRect();
  const cx = rect.left + rect.width/2;
  const cy = rect.top + rect.height/2;
  const relX = (e.clientX - cx) / (rect.width/2);
  const relY = (e.clientY - cy) / (rect.height/2);
  const rotY = relX * maxRot;
  const rotX = relY * -maxRot;
  // use rAF to update css variables
  cancelAnimationFrame(rafId);
  rafId = requestAnimationFrame(() => {
    card.style.setProperty('--rotX', `${rotX}deg`);
    card.style.setProperty('--rotY', `${rotY}deg`);
    // parallax children update similarly based on depth
  });
}

function onLeave() {
  cancelAnimationFrame(rafId);
  card.style.setProperty('--rotX', `0deg`);
  card.style.setProperty('--rotY', `0deg`);
  // reset parallax layers
}


CSS:

.three-d-card {
  transform-style: preserve-3d;
  perspective: 1200px;
  transform: rotateX(var(--rotX, 0)) rotateY(var(--rotY, 0));
  transition: transform 120ms ease-out;
}
