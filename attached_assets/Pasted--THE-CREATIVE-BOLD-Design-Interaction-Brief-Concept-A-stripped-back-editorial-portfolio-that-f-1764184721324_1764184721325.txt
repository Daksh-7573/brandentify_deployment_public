— THE CREATIVE BOLD (Design & Interaction Brief)
Concept

A stripped-back editorial portfolio that foregrounds services, impact, and imagery. Minimal layout, bold typographic hierarchy, and a warm coral accent to create a friendly-but-authoritative look. Meant for designers, photographers, marketers, and other creatives who want a professional, commerce-ready presence.

Visual Language

Tone: Editorial, minimal, service-first, confidently creative

White space: generous margins, breathing layout, centered content

Accent: Coral for CTAs and highlights; monochrome imagery to let projects pop

Color Palette

Paper White (background): #FFFFFF

Ink Black (primary text): #0C0C0C

Mid Gray (muted text): #757575

Coral Accent (primary): #FF6B5A (signature accent for CTAs / badges)

Slate (cards / subtle backgrounds): #F4F6F8

Gold Accent (sparingly): #E2B45F (used only for premium/status states)

Typography

Headings: GT America or Satoshi (clean geometric display)

Subhead / UI: Inter (variable)

Accents: Playfair Display for editorial quotes (optional)

Iconography & Imagery

Icons: single-color stroke icons (coral on hover)

Photography: full-bleed project images, minimal crop; black & white thumbnail option toggle in UI to achieve editorial look

Animation & Motion Principles

Motion is minimal, purposeful, and soft:

Fade + lift for cards (translateY(-6px), opacity anim)

Micro-interaction for CTAs: coral glow + slight scale on hover (0.12s)

Reveal-on-scroll: gentle fade + up (0.3s) with stagger

Image hover: contrast(1.06) and subtle scale

Hero intro: split-text reveal (words fade in sequentially)

Loading state: discreet skeletons with shimmer using coral tint

Accessibility & Performance Guidelines

WCAG AA text contrast; respect prefers-reduced-motion

Only two web fonts (Inter + GT) loaded as variable fonts where possible

Images: srcset, lazy-loading, WebP/AVIF recommendation

PART 2 — FULL PAGE STRUCTURE (Section-by-Section)

Top-level: container max-width 1100–1200px, centered. Sections stacked vertical with clear anchor IDs for nav. Sticky minimal nav (logo + small CTAs).

0. Site Header (sticky, minimal)

Components: Logo (left), Nav links (center or hidden on mobile), CTAs (Book / Media Kit) right

Behavior: Transparent at top, solid white on scroll with subtle shadow

Data: none

1. Hero — Service-First Intro

Goal: Communicate who you are, what you offer, and direct visitors to book or explore services.

Layout: Two columns (desktop) — Left: textual content; Right: key visual / dynamic card carousel.

Data fields

{
  "name","primaryTitle","subTitle","shortIntro","heroServiceList":["Editorial Shoots","Brand Strategy",...],
  "photoURL","ctaPrimary{label,href}","ctaSecondary","socialLinks"
}


Elements

Headline: Name + role line (split reveal, editorial cadence)

Short intro: 1–2 lines (value + service orientation)

Service chips: horizontal pill list (clickable)

Primary CTA (Book) — coral button

Secondary CTA (Media Kit download)

Behavior

Clicking a service chip scrolls to Services or filters projects

Hero visual: dynamic card (rotating work samples), gentle auto-advance, pause on hover

2. Professional Branding (Tagline, About, What I Offer, Looking For)

Goal: Deeper context about the artist's brand and offerings.

Subsections

Tagline (large)

About Me (80–180 words), with Read more toggle

What I Offer: service bullets with short outcomes (e.g., “Runway lookbooks — Attract show bookings”)

Looking For: one-line CTA (e.g., “Looking for brand collaborations & castings”)

Data
tagline, aboutMe, whatIOffer[] (title, benefit), lookingFor

3. Services (Service-First Core)

Goal: Prominent, conversion-focused list of services.

Layout

Card grid (1–3 columns depending on screen)

Each service card: title, 1-line outcome, starting price (optional), key deliverables, CTA “Book Service” / “Enquire”

Data
services[] = {id,title,description,deliverables[],priceRange,iconId}

Behavior

Book opens booking modal with prefilled serviceId

Enquire sends project-specific contact form

4. Highlights / Impact Strip (Quick Stats)

Goal: At-a-glance credibility: campaigns, shows, publications, happy clients.

Elements

3–5 stat tiles (e.g., “120+ shoots”, “20 brands”, “Featured in X”)

Small icon, number, micro-copy

Data
stats[] = {label,value,icon}

5. Projects / Portfolio Gallery (Primary Visual Showcase)

Goal: Show curated work; emphasize results & context.

Layout

Masonry-like grid or uniform cards

Desktop: 3 columns; Tablet: 2; Mobile: 1

Card content: large image, project title, category, year, outcome badge (optional)

Interaction

Click → Project modal (full gallery, story, objective, role, outcomes)

Modal CTAs: “Add to Media Kit”, “Book project-style”

Data
projects[] = {id,title,thumbnailUrl,coverUrl,year,tags,description,mediaUrls[],projectUrl,client}

6. Case Studies (Deep Dives)

Goal: For service-sellers, 2–4 featured case studies with outcomes and process.

Structure

Alternating layout: image left/text right, then reverse

Each case: Challenge → Approach → Outcome (metrics if available)

Data
caseStudies[] = {title,client,year,challenge,approach,results(media),cta}

7. Skills & Tools

Goal: Show craft & tools; visual, quick.

Layout

Horizontal bar list or icon grid with proficiency indicators (Novice→Expert)

Tools tag cloud (e.g., Capture One, Lightroom, Figma)

Data
skills[] = {name,level,proficiencyPct}; tools[] = {name,icon}

8. Experience / Career Timeline

Layout

Minimal vertical timeline: role, brand, dates, one-line highlight

Expand for details (desktop hover / mobile accordion)

Data
experience[] = {title,company,startDate,endDate,location,shortDesc,keyAchievements[]}

9. Education & Training

Layout

Compact list with degree, institution, dates, notable courses/certifications

Data
education[] = {degree,institution,field,startDate,endDate,notes[]}

10. Testimonials & Clients

Layout

Carousel of short testimonials (quote + client name + role + logo)

Optional verification badge

Data
testimonials[] = {quote,clientName,clientTitle,clientLogo,isVerified}

11. Media Kit (Download Modal / Dynamic PDF)

Goal: Compile hero + selected projects into a downloadable PDF media kit.

Behavior

User can generate PDF from profile fields + choose projects (max 6)

Generated server-side or via client PDF builder

Data
Uses other profile fields

12. Contact / Booking (Form)

Fields
name,email,phone,projectType(select),budget(optional),datePref, message, attachFiles

Behavior

Booking modal with available slots (if integrated)

On submit: send to API/CRM + show success toast

CTA also available as email link

13. Footer (Minimal)

Quick links, small bio, social icons, privacy & credits

Implementation Notes & Tips

State & Data flow: keep presentation components stateless. Container components fetch profile JSON and build the HeroData, projects[], etc.

Image optimization: use next/image or equivalent with srcset.

Animations: implement reveal-on-scroll with IntersectionObserver + CSS transitions or Framer Motion. Provide prefers-reduced-motion fallback.

Media Kit: generate server-side for reliability, or use a client PDF library for quick demos.
