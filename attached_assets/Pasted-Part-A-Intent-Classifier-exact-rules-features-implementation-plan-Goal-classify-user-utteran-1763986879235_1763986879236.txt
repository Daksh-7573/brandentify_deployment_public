Part A — Intent Classifier: exact rules, features & implementation plan

Goal: classify user utterances into one of the intent buckets used for follow-up generation:
{clarify, probe, action, resource, confirm, alternative, close}

1) Inputs & engineered features (order for priority)

text_raw — raw user message(s) (use last 1–3 messages concatenated)

profile_summary — user industry/domain, selected brand goals, experience level

dialogue_state_flags — e.g. profile_complete, asked_resume, last_action, platform_pref

available_actions — system actions like generate_post, schedule_call, share_template, connect_person

conversation_history_features:

turn_count (length of conversation)

last_bot_action_type (if bot suggested action)

user_action_response (did user accept/decline)

Derived text features:

token counts, punctuation counts, question mark count

presence of modal verbs (can, should, would)

presence of first/second person pronouns

sentiment score (pos/neg/neutral)

named-entity tags (companies, platforms, skills)

TF-IDF vector or BERT embedding for semantic similarity

2) Keyword & regex rules (fast path) — use these first (deterministic)

Run these in order and if match -> map to the intent. If multiple match, use priority ranking (Action > Clarify > Probe > Resource > Confirm > Alternative > Close).

Action-intent indicators (high priority)

If message contains actionable requests or call-to-action keywords:

Regex / keywords:

\b(draft|write|create|generate|schedule|book|send|share|post|publish|upload|export|download|apply|connect me|introduce me|show me)\b

I'd like you to (draft|write|create|show) or Can you (draft|create|write|post)

help me (create|draft|write|build)

→ Map to action

Clarify-intent indicators

If user asks a clarifying question or a yes/no / binary choice or ambiguous pronouns:

Regex / keywords:

\b(do you mean|which (one|platform)|which (tool|format|version)|did you mean|or do you mean)\b

trailing ? with short question (<10 tokens) and high ambiguity

pronouns with ambiguity: this/that/it/them/they without noun in previous sentence

Patterns:

Short question + profile data lacking (e.g., user says “Do it on LinkedIn?” but last bot message didn't mention LinkedIn explicitly)

→ Map to clarify

Probe-intent indicators (depth & context seeking)

If user expresses goals or partial info and invites exploration:

Keywords:

why, what, how, which, tell me more, explain, more about, go deeper, examples

"I want", "I'm trying to", "my goal is", "I need to"

If message length >12 tokens and contains goal-like words (grow, get clients, pivot, monetize)
→ Map to probe

Resource-intent indicators

When user asks for templates, examples, checklists, tools:

Keywords:

template, example, sample, checklist, outline, templates, pack, tool, guide, resource, swipe file

do you have a template, send me, show sample, give example
→ Map to resource

Confirm-intent indicators

Short confirmations / acceptance language:

Keywords:

yes, yep, sure, okay, do it, please proceed, confirm, sounds good, go ahead, let's do it

If previous bot offered an explicit action and user replies with affirmation -> confirm

Alternative/Reframe indicators

User rejects or asks for different approach:

Keywords:

or, instead, another, different, not that, don’t want, prefer, rather
→ Map to alternative

Close (end-of-conversation)

Keywords:

thanks, thank you, that’s all, bye, done, not now, later
→ Map to close

3) ML layer (fallback + improve coverage)

If deterministic rules don't fire, use a lightweight classifier (fast):

Model: logistic regression or small transformer classifier (DistilBERT) fine-tuned on your labeled intents.

Input features: last message embedding (BERT), token counts, presence of question mark, number of modal verbs, profile flags.

Output: intent probabilities; set threshold 0.45 for top intent; if top probability < 0.45 fallback to probe intent.

4) Post-classification heuristics & tie-breaking

If available_actions include a direct match (e.g., generate_post) and message contains draft or write → force action

If user is short (<8 tokens) and contains ? → prefer clarify

If user is a photographer and mentions portfolio → boost weight for resource/probe about portfolio

If user has goal=Visibility → favor action/resource oriented follow-ups

5) Training labels & data suggestions

Label examples per intent: collect 500–1,000 utterances per class initially.

Domains to sample from: support chat logs, onboarding dialogues, quest interactions, social post creation flows.

Augment data: paraphrase with back-translation and synonyms.

Evaluation: precision/recall per intent; require >85% precision on action and clarify intents (they drive UX).

6) Implementation checklist (engineering)

Implement deterministic keyword/regex filters (fast path).

Run ML classifier only when deterministic rules produce no clear match.

Return intent + confidence + matched_rule so downstream logic knows why.

Log all inputs + chosen intent for review and periodic rule improvements.

Add human-in-the-loop review for ambiguous cases.

Add A/B testing to compare user engagement rates for follow-ups triggered by the new classifier vs old.

Part C — How to use & integrate these templates

Seed DB: Insert templates into followup_templates table with fields: industry, type, text, why, actionHint, tags.

Selection logic:

Filter by industry (if user.profile.industry present). Also keep global templates available for generic fallback.

Score templates by relevance_score = semantic_similarity(user_text, text) + (profile_goal_match ? +0.2) + (dialogue_state_match ? +0.15) + (actionable ? +0.1)

Rerank and pick top 6 candidates; use diversity penalty to remove near-duplicates; return best 3–4 to UI.

Personalization: run simple substitution to inject user variables (e.g., {{platform}}, {{metric}}, {{city}}) — templates are written to allow this. Example: “Shall I draft a 5-slide Instagram carousel showing your design process?” → if user prefers LinkedIn replace “Instagram” with {{platform}}.

Action routing: if template includes actionHint, wire the button to a client route that opens the relevant tool (editor, schedule modal, template viewer).

Feedback loop: log which follow-up was shown and whether user accepted it. Use that to improve ranking weights.
