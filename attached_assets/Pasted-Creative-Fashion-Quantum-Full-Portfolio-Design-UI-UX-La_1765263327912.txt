Creative Fashion Quantum — Full Portfolio Design (UI / UX / Layout / Details)

Below is a complete, production-ready design spec to build the Fashion Quantum Portfolio (card + full-page portfolio). It maps every field to UI, gives layout rules, interaction & animation specs, assets, responsive rules, component props and developer notes — ready for handoff.

1 — Concept Overview

Name: Fashion Quantum Portfolio
Target: Models, Photographers, Stylists, Fashion Creatives
Vibe: Editorial × Minimal × Premium — magazine-like, image-first, soft glows, tasteful accents (no neon).
Primary goals: Showcase visuals, tell the fashion story quickly, convert (bookings / inquiries), look shareable on social.

2 — Design Tokens & Style System
Colors

Noir Black: #050509 (page)

Runway Charcoal: #111118 (section)

Soft Bone: #F5F3EE (text highlights, borders)

Warm Sand: #E2D4C5 (subtle accents)

Blush Pink: #F9C5D5 (primary soft accent / CTA)

Editorial Nude: #D9B99B (secondary accent)

Champagne Glow: #FDF3D9 (glow accent)

Deep Burgundy: #7A1F3D (emphasis)

Ink Grey: #9CA3AF (secondary text)

White: #FFFFFF (primary text)

Type

Display: Playfair Display or Cormorant Garamond (Serif)

UI / Body: Inter or Sora (Sans)

Sizes (desktop): H1 name 34–40px, H2 section 18–24px, body 15–16px, small 11–13px

Letter spacing: headings -0.01em, small caps for chips & labels 0.12em

Spacing & Grid

Page max-width: max-w-6xl (1200px) centered

Columns: CSS grid; desktop 12 / sections use 2–3 columns

Gutter: 24px

Section vertical padding: py-12 (mobile), py-20 (desktop)

Card radius: rounded-2xl (16px)

Elevation & Glass

Card background: rgba(17,17,24,0.7) or #111118 with backdrop-filter: blur(10–16px)

Subtle border: 1px solid rgba(245,243,238,0.12)

Shadow: 0 20px 50px rgba(0,0,0,0.6)

3 — Page Layout (High Level)

Order (scroll top→bottom):

Header (Hero) — card-first, image-forward

About / Snapshot — story + UVP

Skills / Specialties — compact cards

Experience / Brands (timeline)

Education / Training (compact)

Projects / Lookbook (gallery) — hero of visuals

Services / Rates (premium) — booking CTA

Footer contact + social

Layout rules:

Hero uses multi-column: left image / right textual panel on desktop; stacked on mobile.

Projects are image-first masonry grid (2→3 columns).

All sections placed inside glass cards (two-column inside where needed).

4 — Component List & Mapping (fields → UI)

Each component includes required fields (from your list).

4.1 HeroHeader (Header — section 1)

Fields used: name, photoURL, title, tagline, location, industry, domain
Layout:

Full-bleed centered hero with max-width card sitting over background.

Left: Portrait (1:1 or vertical) in framed container (spotlight radial behind).

Right: Name (serif H1), Title (small caps), Tagline (italic or lighter serif), Info chips row (Location, Industry, Domain).

CTA row: primary “View Portfolio” (Blush gradient), secondary “Book Me” (outline), tertiary “Download Comp Card”.

Micro-Interactions:

Portrait: slow parallax on mouse move (desktop) + soft zoom on hover transform: scale(1.03).

Name underline shimmer (animation on load).

Chips: hover lift translateY(-2px) and slight glow.

Accessibility:

Name H1, Title <p> with role; all CTAs keyboard focus styles.

4.2 About / Snapshot (About — section 2)

Fields used: aboutMe, visionStatement, missionStatement, uniqueValueProposition, coreValues[], primaryAudience[], secondaryAudience[], industry, domain, lookingFor, email

Layout:

Two-column card:

Left (2/3): “About Me” text (max 350 words), Vision & Mission micro-cards (stacked).

Right (1/3): UVP highlight (big serif text), Core Values chips, Audiences chips, LookingFor info, email CTA.

UI details:

Vision & Mission cards: left colored stripe (Warm Sand for Vision, Blush for Mission).

UVP: larger serif with soft champagne highlight.

Core Values: pill chips with uppercase text, soft bone border.

Interactions:

UVP: on hover, subtle background highlight fade; chips animate translateX(4px) on hover.

4.3 Skills (Skills — section 3)

Fields used per skill: id, name, proficiency, category, yearsOfExperience

Layout:

Grid (responsive) of SkillCards; each card minimal with:

Skill name, category small label

A linear or circular soft meter (1–5) — use filled circles or 5-segment bar in Blush/Champagne accent.

Years text below.

Design specifics:

Cards: bg: rgba(255,255,255,0.02) with border: rgba(245,243,238,0.06) and small shadow.

Proficiency: accessible label for screen readers.

Interactions:

Hover: lift translateY(-6px) + shadow increase.

4.4 Work Experience (Experience — section 4)

Fields used per experience: id, title, company, industry, domain, location, startDate, endDate, description, keyResponsibilities[]

Layout:

Elegant vertical timeline:

Left vertical line with year markers or nodes.

Right, cards per entry with job title, company, dates, chips for industry/domain, location, short description and 3 responsibilities bullet list.

Current role: subtle champagne glow and animate-ping node.

UI:

Company logos (small) if available (use grayscale, filter: grayscale(100%)).

Date format: Apr 2021 – Present (localized).

Interactions:

Expand/collapse details on click (mobile friendly).

4.5 Education (Education — section 5)

Fields used per education: id, degree, institution, location, fieldOfStudy, industry, startDate, endDate, skillsAcquired[]

Layout:

Compact timeline/cards similar to experience but lighter.

SkillsAcquired displayed as small chips.

Interactions:

Hover reveals skillsAcquired tooltip or inline expansion.

4.6 Projects / Lookbook (Projects — section 6)

Fields used per project: id, title, description, category, technologies[], outcome, startDate, endDate, mediaUrls[], externalUrl, impact, role, teamSize

Layout:

Masonry/justified grid of large image cards.

ProjectCard overlay on hover shows title, role, client, and “View Story” CTA.

Click opens Project Modal with full content: carousel, description, tech/tags, impact metrics, external links.

UI specifics:

Use large images (aspect ratio controlled) with minimal crop; add soft vignette.

Category badge top-left (small caps).

Footer overlays with role & impact metric.

Accessibility:

Modal traps focus; images have alt text.

Interactions:

Image lazy loading; hover overlay fades in (200–300ms).

4.7 Services / What I Offer (Services — section 7 — premium)

Fields used: id, title, description, category, priceInr, priceUsd, isHourly, features[], imageUrl, order, isActive

Layout:

Grid of service cards with price tag.

Each card shows service title, short description, price (INR primary, USD secondary), features bullet list, CTA “Enquire” or “Book”.

Design:

Price highlight bar in Blush/Editorial Nude gradient.

Feature ticks with small icons.

Interactions:

Hover: card grows slightly, price highlight glows.

5 — Visual & Interaction Details (Card & Page)
Card specifics (applies to Hero card and project cards)

Portrait frame: soft spotlight radial behind image (radial-gradient), thin inner border, slight desaturation filter by default, color pop on hover.

Light streak: single slow moving sweep element over hero image (animate like softSweep).

Shimmer underline: linear-gradient animated for name underline (background-position animation).

Subtle Parallax: on mousemove change transform of image and small decorative blobs (translate3d) — limited to desktop.

Motion spec (keyframes)

softSweep — 6–8s ease-in-out infinite for image streak.

shimmerUnderline — 3s linear infinite (only for underline, low opacity).

cardLift — transition all 220–300ms ease for hover lift.

pingCurrent — 1.5s infinite for current role indicator.

Hover states

Buttons: translateY(-2px), box-shadow: 0 14px 32px rgba(253,243,217,0.28).

Cards: translateY(-6px), box-shadow intensified.

Chips: translateX(4px) on hover + subtle background brighten.

6 — Responsive Behavior & Accessibility
Breakpoints

Mobile (≤640px): single column stack, hero image prominent, CTAs stacked.

Tablet (641–1024px): 2 columns for hero/about, projects 2 columns.

Desktop (≥1024px): 2–3 columns, projects 3 columns masonry.

Accessibility

Colors must pass AA for primary text (check contrast with Noir/Runway).

Keyboard focus styles on all CTAs and interactive items.

Aria labels for icons, role="dialog" for modals, aria-live for success toasts.

Images: alt attribute and optional longdesc for gallery.

7 — Assets & Export Guidelines (for designers)

Provide images exported at multiple sizes: 1600px, 1200px, 800px (web optimized).

Provide SVGs for logos, icons; provide Lucide react icon names to implement.

Provide Figma tokens: color swatches, text styles, spacing tokens.

Provide component variants: SkillCard (4 states), ProjectCard (hover), HeroCard (mobile/desktop), ServiceCard (active/inactive).

Provide small Lottie or web-optimized SVG animation for shimmer if using.

8 — Developer Handoff Notes
Components & Props (summary)

HeroHeaderProps { name, photoURL, title, tagline, location, industry, domain, ctaUrls }

AboutProps { aboutMe, visionStatement, missionStatement, uniqueValueProposition, coreValues[], primaryAudience[], secondaryAudience[], industry, domain, lookingFor, email }

SkillsProps { skills: Skill[] }

ExperienceProps { experiences: Experience[] }

EducationProps { educations: Education[] }

ProjectsProps { projects: Project[] }

ServicesProps { services: Service[], isPremium: boolean }

Implementation tips

Use CSS backdrop-filter for glass; provide fallback for browsers without support.

Use IntersectionObserver for entrance animations to avoid animating off-screen.

Use loading="lazy" for all img tags in Projects grid.

Prefer CSS animations for shimmer and sweep; Framer Motion for mount animations where needed.

Respect prefers-reduced-motion.

Performance

Serve images via CDN; use next/image (if Next.js) for automatic optimization.

Avoid animating layout properties; animate transform & opacity only.

9 — Interaction Flows (example UX)

User sees hero → clicks View Portfolio

Smooth scroll to Projects (use scrollIntoView({behavior:'smooth'})).

User clicks Project

Modal opens; carousel starts at selected image; focus trapped; close returns focus to trigger.

User clicks Book Me

Open contact modal prefilled with name field if user logged in; secondary flows: open email mailto: fallback.

10 — Sample Content Guidelines (copywriting)

Name: big serif — e.g., “Ananya Verma”

Title: small caps — “MODEL · EDITORIAL · RUNWAY”

Tagline: one line — “Turning concepts into frames.”

About (max 350 words): lead sentence + 2 short paras + 1 CTA sentence

UVP: one sentence (max 150 chars) — e.g., “I bring high-fashion poise and campaign reliability to editorial shoots.”
